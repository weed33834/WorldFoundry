## Current Owner

The canonical LVDM package is `worldfoundry/base_models/diffusion_model/video/lvdm`.
Runtime configs should point to fully qualified `worldfoundry.base_models.diffusion_model.video.lvdm...`
targets instead of short `lvdm.*` imports or model-specific fork paths.

## Guard Rails

Consolidate one fork at a time. For each fork:

1. Move fork-only modules under `video/lvdm/variants/<model-id>` or merge them into the canonical module when the APIs are identical.
2. Rewrite imports and YAML targets to full `worldfoundry...video.lvdm...` paths.
3. Delete the old model-specific `lvdm` directory.
4. Run `py_compile` or `compileall`, target import checks, and the relevant `test/eval_core/test_foundation_model_integration.py` subset.

Do not change Dynamicrafter, MotionCtrl, or Pandora canonical paths unless a focused regression proves it is required.
