## 介绍

验证证据应和改动 surface 成比例：足够小，可以本地运行；足够具体，可以抓到回归；对跳过的重型阶段要明确说明。

## 按变更类型选择证据

| 变更 | 最小证据 | 提升状态时的额外证据 |
| --- | --- | --- |
| 只改文档 | `npm run types:check`、`npm run build`、受影响 route 返回 200 | 布局或交互变化需要浏览器截图。 |
| CLI 或 runner | 聚焦 `test/eval_core` 测试加代表性 evaluate 命令 | 如果输出契约变化，再跑 existing-results 和 model-mode runs。 |
| Model manifest | `zoo model-show`、`zoo model-download --check-local` | Clone、checkpoint、demo parity、runner parity、benchmark run。 |
| Model runtime integration | 仓内 runtime import、无机器本地 source checkout 依赖、带 `backend_stage` 的 runtime profile，以及聚焦回归 | Checkpoint-backed run 和一份 normalized benchmark scorecard。 |
| Benchmark manifest | `zoo benchmark-show`、`preflight runtime --profile <benchmark-id>` | Clone、dataset、official runtime、normalized scorecard。 |
| Benchmark runner | `zoo benchmark-run --mode official-validation` 或 `--mode official-run` | Official result import、official runtime 和 full-suite fairness gate。 |
| Suite 或 matrix | `run --all-benchmarks --plan-only --json` | 规划选中的 cells，然后用可运行 benchmark 路径验证 artifacts。 |

## 产物验收

| 产物 | 必需场景 | 通过标准 |
| --- | --- | --- |
| `run_manifest.json` | 任意 evaluation run | Command、ids、status、timestamps、counts 和 output paths 与 run 一致。 |
| `execution_plan.json` | Runner 或 suite 变化 | Resolved model、benchmark、metrics 和 requests 是确定性的。 |
| `sample_ledger.jsonl` | 部分失败 review | Failed/skipped samples 有可见原因。 |
| `metrics/summary.json` | Metric 变化 | Aggregates、failed counts 和 metric keys 符合预期 schema。 |
| `scorecard.json` | 可发布 evaluation evidence | Eligibility、leaderboard state、metrics 和 blockers 存在。 |
| `report.json` | Model/benchmark validation | Stage status、command、repo/checkpoint/dataset refs 和 blockers 已记录。 |
| `suite_report.json` | Suite 或 matrix 变化 | 每个选中 cell 都是 pass、fail 或 skipped，并带原因。 |

## 集成声明规则

`integrated` 只用于可 review 的已实现 support surface，不能只靠 source URL 或 plan 声明。对模型 runtime，模型架构或 runtime glue、启动路径、环境契约、checkpoint/input/output contract，以及最小真实推理证据，都必须能从仓库和声明资产中 review。对 benchmark，仓内 runner、official command、隔离 importer 或 official-result normalizer 只要 CLI 路径、artifact/data requirements 和 scorecard 输出都有证据，就可以是 `integrated`。

不要把 `integrated` 当成 leaderboard 同义词。Normalizer-only benchmark integration 必须保持 `verification_status=normalizer_only` 和 `leaderboard_valid=false`，直到完整 official scorer 或 judge 已运行，且具体 scorecard 证明 eligibility。

## 失败策略

| 状态 | 含义 | 发布处理 |
| --- | --- | --- |
| Pass | 必需阶段已运行，并产出预期产物。 | 可以支撑对应 claim。 |
| Partial | 有阶段因为 network、credentials、data、GPU 或 license review 被跳过。 | 可用于开发，不能支撑更强公开 claim。 |
| Blocked | 必需 source、dataset、license、dependency、checkpoint 或 API 条件不可用。 | Manifest 保持 blocked 或 planned。 |
| Stale | Report hash、manifest hash、git commit、validator script 或 timestamp 不再匹配。 | 重新运行验证后才能计入。 |
| Runner-only | Schema 和 runner path 通过，但 official runtime 未运行。 | 不是 leaderboard evidence。 |

## Validation matrix

Matrix 是 model-zoo 和 benchmark-zoo readiness 的 release gate。它会合并 manifest entries 和 staged validation reports，并重新计算 integration status。

```bash
conda run -p "${WORLDFOUNDRY_UNIFIED_ENV_PREFIX}" worldfoundry-eval run \
  --all-benchmarks \
    --plan-only \
  --output-dir tmp/worldfoundry_plan_matrix \
  --json

conda run -p "${WORLDFOUNDRY_UNIFIED_ENV_PREFIX}" worldfoundry-eval validate-artifact \
  tmp/worldfoundry_plan_matrix/suite_manifest.json \
  --kind suite-manifest \
  --strict \
  --json
```

| Gate status | 含义 | Maintainer action |
| --- | --- | --- |
| `accepted_integrated` | Fresh report 通过所有必需阶段，manifest 也声明 integrated。 | 保持证据 fresh。 |
| `accepted_variant_integrated` | 一个或多个 scoped variants 通过，但 parent family 仍是 planned。 | 继续验证剩余 variants。 |
| `validated_but_manifest_not_promoted` | Report 通过，但 manifest 仍是 planned 或 blocked。 | Artifact review 后再提升。 |
| `manifest_integrated_without_report` | Manifest 声称 integrated，但没有 fresh validation report。 | 运行 staged validation 或降级。 |
| `manifest_overclaims_integration` | Manifest 声称 integrated，但 computed stages 不通过。 | 降级或重新运行 full validation。 |
| `report_overclaims_integration` | Report 声称 integrated，但 staged evidence 不完整。 | 丢弃或重跑 report。 |
| `stale_report` | Report hash、script、git commit、schema 或 manifest entry 改变。 | 重新运行验证。 |
| `partial_or_blocked` | Report 存在，但 stages 失败或被跳过。 | 按 `next_action` 处理。 |
| `not_validated` | Manifest entry 没有 report。 | 运行对应 runtime preflight 和 official validation run。 |
