## Environment setup

### Runtime environment

- **Conda env:** `worldfoundry-unified-cu128`.
- **Separate setup:** defaults to the unified WorldFoundry env (`needs_new_env: false`); a benchmark-only conda env is usually not required.
- WorldFoundry provides contract and official result normalization for caller-provided RoboCerebra outputs.

### Evaluation data

- Candidate model outputs: set `WORLDFOUNDRY_ROBOCEREBRA_RESULTS_PATH` to the generated-video or rollout artifact root.

## Run Evaluation



After preparing the assets and candidate artifacts listed on this page, use the public run entry points below.

### Set Candidate Artifacts

```bash
export WORLDFOUNDRY_GENERATED_ARTIFACT_DIR=/path/to/generated/artifacts
```

### Import Official Results

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id robocerebra \
  --mode normalizer \
  --official-results-path /path/to/official/results-or-report \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/robocerebra/normalizer \
  --json
```

### Score Generated Artifacts

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id robocerebra \
  --mode official-run \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/robocerebra/official-run \
  --json
```

### Direct In-Tree Runner

```bash
worldfoundry-eval zoo benchmark-run --benchmark-id robocerebra --mode official-validation --official-results-path <official_results.json> --output-dir <out> --json
```
