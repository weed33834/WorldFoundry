## 环境准备

### 运行环境

- **Conda 环境：** `worldfoundry-unified-cu128`。
- **单独配置：** 默认复用 WorldFoundry 统一环境（`needs_new_env: false`），通常不需要额外 benchmark-only conda env。
- WorldFoundry provides contract and official result normalization for caller-provided RoboCerebra outputs.

### 测评数据

- 待测模型输出目录：设置 `WORLDFOUNDRY_ROBOCEREBRA_RESULTS_PATH` 指向生成视频或 rollout artifact 根目录。
- `${WORLDFOUNDRY_DATA_DIR}/datasets/robocerebra`
- `${WORLDFOUNDRY_CACHE_DIR}/repos/buaa-colalab--RoboCerebra`

### Checkpoint 与资产

- 该 benchmark 以官方结果归一化或仿真 rollout 为主；policy / simulator checkpoint 见 catalog `requires` 与 task YAML。

### 关键环境变量

- `WORLDFOUNDRY_ROBOCEREBRA_ROOT`
- `WORLDFOUNDRY_ROBOCEREBRA_RESULTS_PATH`



## 运行评测



准备好该 benchmark 页面列出的资产和候选产物后，使用下面的公开运行入口。

### 设置候选产物目录

```bash
export WORLDFOUNDRY_GENERATED_ARTIFACT_DIR=/path/to/generated/artifacts
```

### 导入已有官方结果

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id robocerebra \
  --mode normalizer \
  --official-results-path /path/to/official/results-or-report \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/robocerebra/normalizer \
  --json
```

### 从生成产物重新打分

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id robocerebra \
  --mode official-run \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/robocerebra/official-run \
  --json
```

### 直接调用仓内 runner

```bash
worldfoundry-eval zoo benchmark-run --benchmark-id robocerebra --mode official-validation --official-results-path <official_results.json> --output-dir <out> --json
```
