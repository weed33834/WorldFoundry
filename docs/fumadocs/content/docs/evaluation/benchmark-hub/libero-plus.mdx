## Environment setup

### Runtime environment

- **Conda env:** `libero-plus-runtime`.
- **Separate setup:** this benchmark expects a dedicated conda env rather than the unified runtime.
- LIBERO and LIBERO-Plus package constraints may conflict; use a separate environment if the unified runtime cannot satisfy both.

### Evaluation data

- Candidate model outputs: set `WORLDFOUNDRY_LIBERO_PLUS_RESULTS_PATH` to the generated-video or rollout artifact root.
- `${WORLDFOUNDRY_DATA_DIR}/datasets/libero-plus`
- `${WORLDFOUNDRY_CACHE_DIR}/repos/Lifelong-Robot-Learning--LIBERO`

### Checkpoints & assets

- This benchmark is primarily rollout- or official-result-driven; see catalog `requires` and the task YAML for policy/simulator checkpoints.

## Run Evaluation



After preparing the assets and candidate artifacts listed on this page, use the public run entry points below.

### Set Candidate Artifacts

```bash
export WORLDFOUNDRY_GENERATED_ARTIFACT_DIR=/path/to/generated/artifacts
```

### Import Official Results

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id libero-plus \
  --mode normalizer \
  --official-results-path /path/to/official/results-or-report \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/libero-plus/normalizer \
  --json
```

### Score Generated Artifacts

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id libero-plus \
  --mode official-run \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/libero-plus/official-run \
  --json
```

### Direct In-Tree Runner

```bash
worldfoundry-eval zoo benchmark-run --benchmark-id libero-plus --mode official-validation --official-results-path <official_results.json> --output-dir <out> --json
```
