## 环境准备

### 运行环境

- **Conda 环境：** `kinetix-jax`。
- **单独配置：** 需要 benchmark 专用 conda 环境，不要与统一 runtime 混用。
- Keep Kinetix in a JAX-compatible simulator environment when the unified PyTorch runtime is insufficient.

### 测评数据

- 待测模型输出目录：设置 `WORLDFOUNDRY_KINETIX_RESULTS_PATH` 指向生成视频或 rollout artifact 根目录。
- `${WORLDFOUNDRY_DATA_DIR}/datasets/kinetix`
- `${WORLDFOUNDRY_CACHE_DIR}/repos/FLAIROx--Kinetix`

### Checkpoint 与资产

- 该 benchmark 以官方结果归一化或仿真 rollout 为主；policy / simulator checkpoint 见 catalog `requires` 与 task YAML。

### 关键环境变量

- `WORLDFOUNDRY_KINETIX_ROOT`
- `WORLDFOUNDRY_KINETIX_RESULTS_PATH`



## 运行评测



准备好该 benchmark 页面列出的资产和候选产物后，使用下面的公开运行入口。

### 设置候选产物目录

```bash
export WORLDFOUNDRY_GENERATED_ARTIFACT_DIR=/path/to/generated/artifacts
```

### 导入已有官方结果

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id kinetix \
  --mode normalizer \
  --official-results-path /path/to/official/results-or-report \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/kinetix/normalizer \
  --json
```

### 从生成产物重新打分

```bash
worldfoundry-eval zoo benchmark-run \
  --benchmark-id kinetix \
  --mode official-run \
  --generated-artifact-dir "${WORLDFOUNDRY_GENERATED_ARTIFACT_DIR}" \
  --output-dir tmp/kinetix/official-run \
  --json
```

### 直接调用仓内 runner

```bash
worldfoundry-eval zoo benchmark-run --benchmark-id kinetix --mode official-validation --official-results-path <official_results.json> --output-dir <out> --json
```
