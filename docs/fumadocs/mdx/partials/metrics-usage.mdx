## How to use each metric

Three steps:

1. `source WorldFoundry/tmp/worldfoundry_unified_env.sh` and `pip install -e ".[distribution_metrics]"`
2. `worldfoundry-eval metric show <id>` to resolve ids and aliases
3. Copy the recipe below for your metric and run it

Only **built-in existing-results** metrics use `worldfoundry-eval evaluate`; everything else is Python (`compute_*` or scorer classes).

<MetricQuickNav locale="en" />

Metric imports are lazy: checkpoints are required only when a metric loads a feature extractor, multimodal scorer, or API judge. Default caches follow Hugging Face / Torch / torchvision layout:

```bash
export WORLDFOUNDRY_HFD_ROOT=${HF_HOME:-$HOME/.cache/huggingface}
export WORLDFOUNDRY_T2V_METRICS_CACHE_DIR=$WORLDFOUNDRY_HFD_ROOT
export WORLDFOUNDRY_CKPT_DIR=$HOME/.cache/worldfoundry/checkpoints
```


### `artifact_count` (↑)

**When**: You already ran inference and have a `GenerationResult` JSONL.

**Reference:** Counts output artifacts present on each generation result.

**Assets:** No model checkpoint — `GenerationResult` JSON/JSONL only.

```bash
worldfoundry-eval evaluate \
  --mode existing-results \
  --results-path runs/results.jsonl \
  --output-dir runs/eval/artifact_count \
  --metric artifact_count \
  --json
```

**Returns**: Scorecard JSON under `--output-dir` with per-row artifact counts.

### `required_artifacts_present` (↑)

**When**: Check that every result contains required output artifacts.

**Reference:** Returns 1 when every required artifact name is present, else 0.

**Assets:** No model checkpoint — `GenerationResult` JSON/JSONL only.

```bash
worldfoundry-eval evaluate \
  --mode existing-results \
  --results-path runs/results.jsonl \
  --output-dir runs/eval/required \
  --required-artifact generated_artifact \
  --required-artifact video \
  --metric required_artifacts_present \
  --json
```

**Returns**: `1` when all `--required-artifact` names exist on a row, else `0`.

### `has_artifact:<name>` (↑)

**When**: Parameterized presence check for one artifact name.

**Reference:** Parameterized check: returns 1 when a named artifact exists on the result.

**Assets:** No model checkpoint — `GenerationResult` JSON/JSONL only.

```bash
worldfoundry-eval evaluate \
  --mode existing-results \
  --results-path runs/results.jsonl \
  --output-dir runs/eval/has_video \
  --metric has_artifact:video \
  --json
```

**Returns**: `1` if artifact `video` exists on the row.

### `numeric / numeric:<name>` (↑)

**When**: Emit numeric fields already stored on each result row.

**Reference:**
- **`numeric`:** Emits all numeric metrics, scores, and timings found on a result.
- **`numeric_value`:** Emits one named numeric field from result metadata (e.g. reward).

**Assets:**
- **`numeric`:** No model checkpoint — `GenerationResult` JSON/JSONL only.
- **`numeric_value`:** No model checkpoint — `GenerationResult` JSON/JSONL only.

```python
# all numeric fields
worldfoundry-eval evaluate ... --metric numeric

# one field, e.g. reward from model metadata
worldfoundry-eval evaluate ... --metric numeric:reward
```

**Returns**: Named scalar(s) copied from result metadata/scores.

### `clip_score` (↑)

**When**: Score image–text alignment with CLIPScore.

**Reference:** Cosine similarity between CLIP image and text embeddings. Paper: [CLIPScore](https://arxiv.org/abs/2104.08718) Repo: [linzhiqiu/t2v_metrics](https://github.com/linzhiqiu/t2v_metrics)

**Assets:** OpenCLIP / BLIP2 / PickScore / InternVideo2 / UMT weights via `WORLDFOUNDRY_T2V_METRICS_CACHE_DIR` → `WORLDFOUNDRY_HFD_ROOT` → package `hf_cache`.

```python
from worldfoundry.evaluation.tasks.metrics import CLIPScore, list_all_clipscore_models

print(list_all_clipscore_models()[:5])

clip = CLIPScore(model="openai:ViT-L/14", device="cuda")
score = clip.forward(images="sample.png", texts="a red sports car on a wet street")
print(float(score.item()))
```

**Returns**: Float tensor; higher = better alignment.

### `vqa_score` (↑)

**When**: Probability a VQA model answers yes to a text probe about the image.

**Reference:** Probability a VQA model answers "yes" to "Does this figure show `{text}`?" Paper: [VQAScore (ECCV 2024)](https://arxiv.org/abs/2404.01291) Repo: [linzhiqiu/t2v_metrics](https://github.com/linzhiqiu/t2v_metrics)

**Assets:** Hugging Face / LAVIS / WorldFoundry base-model caches for local VQA models; API models need provider credentials (`OPENAI_API_KEY`, Gemini key, etc.).

```python
from worldfoundry.evaluation.tasks.metrics import VQAScore

vqa = VQAScore(model="clip-flant5-xxl", device="cuda")
score = vqa.forward(
    images="sample.png",
    texts="Is there a dog wearing sunglasses in this image?",
)
print(float(score.item()))
```

**Returns**: Probability in `[0, 1]`; higher = better.

### `itm_score` (↑)

**When**: Image–text matching (ITM) for images or videos.

**Reference:** Image-text matching score from BLIP-2 / InternVideo2 ITM backbones. Paper: [BLIP-2](https://arxiv.org/abs/2301.12597) Repo: [linzhiqiu/t2v_metrics](https://github.com/linzhiqiu/t2v_metrics)

**Assets:** Same cache order as CLIPScore. BLIP2/LAVIS URLs, InternVideo2 `internvl_c_13b_224px`, UMT `b16_ptk710_f8_res224` or `l16_ptk710_f8_res224`.

```python
from worldfoundry.evaluation.tasks.metrics import ITMScore

itm = ITMScore(model="blip2-itm", device="cuda")
score = itm.forward(images="sample.png", texts="a cat sitting on a windowsill")

```

**Returns**: Matching score tensor; higher = better match.

### `gme_score / nexus_score / natural_score` (↑)

**When**: OpenS2V subject-to-video evaluation with folder layout + JSON manifest.

**Reference:**
- **`gme_score`:** GME-Qwen2-VL image-text alignment for subject-to-video evaluation. Paper: [OpenS2V-Nexus](https://arxiv.org/abs/2505.20292) Repo: [PKU-YuanGroup/OpenS2V-Nexus](https://github.com/PKU-YuanGroup/OpenS2V-Nexus)
- **`nexus_score`:** YOLO-World + GME subject phrase-matching success score. Paper: [OpenS2V-Nexus](https://arxiv.org/abs/2505.20292) Repo: [PKU-YuanGroup/OpenS2V-Nexus](https://github.com/PKU-YuanGroup/OpenS2V-Nexus)
- **`natural_score`:** GPT API naturalness rating for generated subject-to-video clips. Paper: [OpenS2V-Nexus](https://arxiv.org/abs/2505.20292) Repo: [PKU-YuanGroup/OpenS2V-Nexus](https://github.com/PKU-YuanGroup/OpenS2V-Nexus)

**Assets:**
- **`gme_score`:** `WORLDFOUNDRY_GME_MODEL_PATH` or `Alibaba-NLP/gme-Qwen2-VL-7B-Instruct/` Hugging Face snapshot.
- **`nexus_score`:** `WORLDFOUNDRY_YOLO_WORLD_CKPT`, `WORLDFOUNDRY_YOLO_CLIP_MODEL` (`openai/clip-vit-base-patch32`), and `WORLDFOUNDRY_GME_MODEL_PATH` for offline runs.
- **`natural_score`:** No local checkpoint — OpenAI-compatible judge API (`OPENAI_API_KEY` or `api_key=...`).

```python
from worldfoundry.evaluation.tasks.metrics import (
    compute_gme_score,
    compute_nexus_score,
    compute_natural_score,
)

manifest = "opens2v/eval.json"  # keyed by video stem
videos = "opens2v/videos"
refs = "opens2v/ref_images"

gme = compute_gme_score(
    input_video_folder=videos,
    input_image_folder=refs,
    input_json_file=manifest,
    device="cuda",
)
nexus = compute_nexus_score(
    input_video_folder=videos,
    input_image_folder=refs,
    input_json_file=manifest,
    device="cuda",
)
natural = compute_natural_score(
    input_video_folder=videos,
    input_json_file=manifest,
)  # needs OPENAI_API_KEY
print(gme, nexus, natural)
```

**Returns**: Dict payloads per video; see OpenS2V eval JSON schema.

Manifest entry example: `{"video_001": {"img_paths": [...], "class_label": [...]}}`.

### `fid` (↓)

**When**: Compare two folders of images (reference vs generated).

**Reference:** Fréchet distance between Inception (or CLIP/SwAV) feature distributions. Paper: [Heusel et al.](https://arxiv.org/abs/1706.08500) Repo: [toshas/torch-fidelity](https://github.com/toshas/torch-fidelity)

**Assets:** Torch-fidelity InceptionV3 / CLIP / SwAV weights via `torch.hub` and `torchvision` caches. Offline: `feature_extractor_weights_path` or staged Torch cache.

```python
from worldfoundry.evaluation.tasks.metrics import compute_fid

ref_dir = "/data/coco_val"
gen_dir = "/data/my_model_samples"

fid = compute_fid(ref_dir, gen_dir)
print(fid)  # lower is better

clip_fid = compute_fid(ref_dir, gen_dir, feature_extractor="clip-vit-b-32")
swav_fid = compute_fid(ref_dir, gen_dir, feature_extractor="swav-resnet50")
```

**Returns**: Float FID; lower is better.

### `fwd / cmmd / clean_fid / mind / trend` (↓)

**When**: Distribution distance on image folders (different feature backends).

**Reference:**
- **`fwd`:** Fréchet Wavelet Distance between image distributions. Paper: [Karras et al.](https://arxiv.org/abs/1812.04948) Repo: WorldFoundry (vendored `pytorchfwd`)
- **`cmmd`:** CLIP Maximum Mean Discrepancy between image sets. Paper: [Jayasumana et al.](https://arxiv.org/abs/2402.05236) Repo: WorldFoundry (adapted from CMMD repo)
- **`clean_fid`:** Clean-FID with improved Inception preprocessing. Paper: [Parmar et al.](https://arxiv.org/abs/2204.06191) Repo: [GaParmar/clean-fid](https://github.com/GaParmar/clean-fid)
- **`mind`:** Monge Inception Distance between image sets. Paper: [John et al.](https://arxiv.org/abs/2306.04594) Repo: WorldFoundry (torch-fidelity integration)
- **`trend`:** Truncated entropy / trend divergence on feature histograms. Paper: [Han et al.](https://arxiv.org/abs/2206.08599) Repo: WorldFoundry (paper reimplementation)

**Assets:**
- **`fwd`:** Torch-fidelity / wavelet backend weights via vendored distribution helpers and torch cache.
- **`cmmd`:** CLIP vision encoder `openai/clip-vit-large-patch14-336` in Hugging Face `transformers` cache (`WORLDFOUNDRY_HFD_ROOT`).
- **`clean_fid`:** Clean-FID Inception network and optional reference statistics in the `cleanfid` runtime cache.
- **`mind`:** Torch-fidelity feature extractors via `torch.hub` / `torchvision` cache.
- **`trend`:** No model checkpoint when caller supplies the required statistics/features.

```python
from worldfoundry.evaluation.tasks.metrics import (
    compute_fwd,
    compute_cmmd,
    compute_clean_fid,
    compute_mind,
    compute_trend,
)

ref, gen = "/data/ref", "/data/gen"
print("fwd", compute_fwd(ref, gen))
print("cmmd", compute_cmmd(ref, gen))
print("clean_fid", compute_clean_fid(ref, gen, dataset_name="cifar10", mode="clean"))
print("mind", compute_mind(ref, gen))
print("trend", compute_trend(ref, gen))
```

**Returns**: Float or dict per metric; lower is better for these ids.

### `vendi_score / rke / rnd` (↑)

**When**: Diversity from feature matrices or image folders.

**Reference:**
- **`vendi_score`:** Vendi score (diversity) from feature matrix or image embeddings. Paper: [Friedman & Dieng](https://arxiv.org/abs/2202.03269) Repo: WorldFoundry (adapted)
- **`rke`:** Rényi kernel entropy mode count on feature embeddings (diversity). Paper: [Jalali et al. (NeurIPS 2023)](https://openreview.net/forum?id=PdZhf6PiAb) Repo: [mjalali/renyi-kernel-entropy](https://github.com/mjalali/renyi-kernel-entropy)
- **`rnd`:** Random Network Distillation novelty score on feature vectors. Paper: [Burda et al.](https://arxiv.org/abs/1810.12894) Repo: WorldFoundry (paper reimplementation)

**Assets:**
- **`vendi_score`:** Image helper: torchvision InceptionV3. Text helper: `roberta-base` and `princeton-nlp/unsup-simcse-roberta-base` under `WORLDFOUNDRY_HFD_ROOT`. Feature-array mode needs no extractor.
- **`rke`:** No model checkpoint when caller supplies feature embeddings.
- **`rnd`:** No model checkpoint when caller supplies feature embeddings.

```python
import numpy as np
from worldfoundry.evaluation.tasks.metrics import (
    compute_vendi_score,
    compute_rke,
    compute_rrke,
    compute_rnd,
    compute_rnd_from_images,
)

features = np.load("embeddings.npy")  # shape (N, D)
print("vendi", compute_vendi_score(features))
print("rke", compute_rke(features))
print("rrke", compute_rrke(ref_features, gen_features))
print("rnd", compute_rnd(features))
print("rnd_images", compute_rnd_from_images("/data/generated"))
```

**Returns**: Float diversity scores; higher vendi/rke/rnd = more diverse.

### `fld / multimodal_mid / fjd / crosslid / cfid / ssd` (mixed)

**When**: Feature-array metrics for fidelity, multimodal alignment, or diversity.

**Reference:**
- **`fld`:** Feature Likelihood Divergence between real and generated feature distributions. Paper: [Richter & Rhodes (ICML 2022)](https://arxiv.org/abs/2204.06496) Repo: WorldFoundry (paper reimplementation)
- **`multimodal_mid`:** Mutual Information Divergence comparing joint image-text embedding distributions. Paper: [Dhaka et al. (NeurIPS 2021)](https://arxiv.org/abs/2106.04566) Repo: WorldFoundry (paper reimplementation)
- **`fjd`:** Fréchet Joint Distance over paired image-text embeddings. Paper: [Lee et al. (ICCV 2023)](https://arxiv.org/abs/2303.08118) Repo: WorldFoundry (paper reimplementation)
- **`crosslid`:** Cross local intrinsic dimensionality for diversity assessment. Paper: [Barua et al. (ICLR 2019)](https://openreview.net/forum?id=H1x0sA4tDr) Repo: [sukarnabarua/CrossLID](https://github.com/sukarnabarua/CrossLID)
- **`cfid`:** Conditional FID from paired conditional/unconditional embeddings. Paper: [Benny & Wolf (NeurIPS 2020)](https://arxiv.org/abs/2008.06993) Repo: WorldFoundry (paper reimplementation)
- **`ssd`:** Semantic Similarity Distance between text and image embeddings. Paper: [Tschannen et al. (ICLR 2018)](https://arxiv.org/abs/1710.05135) Repo: WorldFoundry (paper reimplementation)

**Assets:**
- **`fld`:** No model checkpoint when caller supplies train/test/generated feature arrays.
- **`multimodal_mid`:** No model checkpoint when caller supplies paired feature arrays.
- **`fjd`:** No model checkpoint when caller supplies joint embeddings.
- **`crosslid`:** No model checkpoint when caller supplies feature arrays.
- **`cfid`:** No model checkpoint when caller supplies conditional feature arrays.

```python
import numpy as np
from worldfoundry.evaluation.tasks.metrics import (
    compute_fld,
    compute_multimodal_mid,
    compute_fjd_from_joint_embeddings,
    compute_crosslid,
    compute_cfid,
    compute_ssd,
)

train = np.load("train_feats.npy")
test = np.load("test_feats.npy")
gen = np.load("gen_feats.npy")
print("fld", compute_fld(train, test, gen))
print("mid", compute_multimodal_mid(image_feats, text_feats))
print("fjd", compute_fjd_from_joint_embeddings(real_joint, gen_joint))
print("crosslid", compute_crosslid(features, k=5))
print("cfid", compute_cfid(ref_pairs, gen_pairs))
print("ssd", compute_ssd(text_embeds, image_embeds))
```

**Returns**: Float or dict; check each function docstring for direction.

### `attribute_sad / attribute_pad` (↓)

**When**: SadPaD attribute divergence from precomputed HCS histograms.

- **`attribute_pad`:** No model checkpoint — pass precomputed HCS histograms (`hcs_real`, `hcs_gen`, `text_list`).

### `lpips / ssim / ms_ssim / psnr` (mixed)

**When**: Pairwise reconstruction / consistency on aligned image pairs.

**Reference:**
- **`lpips`:** Learned Perceptual Image Patch Similarity (LPIPS). Paper: [Zhang et al.](https://arxiv.org/abs/1801.03924) Repo: [richzhang/PerceptualSimilarity](https://github.com/richzhang/PerceptualSimilarity)
- **`ssim`:** Structural Similarity Index. Paper: [Wang et al.](https://ieeexplore.ieee.org/document/1284395) Repo: WorldFoundry (PIQ/torchmetrics)
- **`ms_ssim`:** Multi-scale SSIM. Paper: [Wang et al.](https://ieeexplore.ieee.org/document/1292216) Repo: WorldFoundry (PIQ/torchmetrics)
- **`psnr`:** Peak Signal-to-Noise Ratio. Repo: WorldFoundry (PIQ/torchmetrics)

**Assets:**
- **`lpips`:** VGG16 and LPIPS / DISTS / PieAPP auxiliary weights via torchvision and torch hub release URLs.
- **`ssim`:** No model checkpoint — aligned image arrays only.
- **`ms_ssim`:** No model checkpoint — aligned image arrays only.

```python
import numpy as np
from PIL import Image
from worldfoundry.evaluation.tasks.metrics import (
    compute_lpips,
    compute_ssim,
    compute_ms_ssim,
    compute_psnr,
    compute_perceptual_bundle,
)

ref = np.array(Image.open("ref.png").convert("RGB"))
gen = np.array(Image.open("gen.png").convert("RGB"))

print("lpips", compute_lpips(ref, gen))          # lower better
print("ssim", compute_ssim(ref, gen))            # higher better

print("psnr", compute_psnr(ref, gen))
print("bundle", compute_perceptual_bundle(ref, gen))
```

**Returns**: Float per metric; LPIPS lower, SSIM/MS-SSIM/PSNR higher is better.

### `dino_similarity / dreamsim / fsim / cpbd` (mixed)

- **`fsim`:** Feature Similarity Index for color images. Paper: [Zhang et al.](https://ieeexplore.ieee.org/document/5705575) Repo: WorldFoundry (PIQ/torchmetrics)

**Assets:**

- **`dreamsim`:** DreamSim release ZIPs from `ssundaram21/dreamsim`; pass `cache_dir=...` to `compute_dreamsim(...)`.
- **`fsim`:** No model checkpoint — aligned image arrays only.

```python
import numpy as np
```

**Returns**: Float scores; DreamSim lower = more similar.
